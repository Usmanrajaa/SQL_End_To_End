5.
You DON’T just use RENAME TABLE (causes disruption)

-- Strategy:
Create new DB
CREATE DATABASE company_db_v2;
-- Copy schema
CREATE TABLE company_db_v2.employees LIKE company_db.employees;
-- Copy data
INSERT INTO company_db_v2.employees
SELECT * FROM company_db.employees;
Sync ongoing changes (critical step)
Use triggers / binlog replication
Switch application connection to new DB
Drop old DB later
